# Changelog

Notable changes to MECE Studio. Newest first. (Open items live in `NEXT_STEPS.md`.)

## [Unreleased]

### Fixed
- **Dense trees no longer overlap.** A value-driver node showing a value, evidence, and ME/CE dots renders taller than a bare box; the layout now feeds each node's real (content-derived) height to dagre, so a tall node no longer overruns the gap into its siblings. Simple trees lay out exactly as before.

### Added
- **Rubber-band (box) select.** Hold **Shift and drag** a box on empty canvas to select every node inside it (a plain drag still pans) — the same floating action bar then sets status / priority or deletes them in one undoable step. Selection stays store-owned (the ring is the store's), so pan, drag-to-reparent, and the keyboard outliner are untouched; React Flow owns the pointer gesture and syncs the result back through a single guarded `onSelectionChange`. Completes the F13 multi-select work (⌘/Ctrl/Shift-click already shipped).
- **PowerPoint decks are now editable.** **Export ▾ → PowerPoint** renders the tree as **native slide objects** — a rounded box + label per node (carrying its status colour, value, priority, evidence and ME/CE state) and a connector line per branch, placed from the exact on-canvas layout — so you can move, restyle, and edit nodes in PowerPoint instead of getting a flat screenshot. Very large trees (over ~150 nodes) still fall back to the embedded-image slide. (`export-safety` note: every node string goes through pptxgenjs `addText`, which escapes it into inert Office Open XML — no live markup.)
- **Edit evidence in place.** An evidence item's text is now editable inline in the inspector's **Evidence** tab — click it, type, and it commits on **blur / Enter** (one undo step), alongside the existing strength and supports/contradicts controls. (An unchanged or blank edit is a no-op, so it never churns undo.)
- **Add a sub-issue from the node.** Selecting or hovering a node now shows a **＋** on its child edge — click it to add a sub-issue and drop straight into editing its label, the same gesture as pressing **Tab**, without reaching for the keyboard or the inspector.
- **Mixed-axis warning.** The MECE review now flags a classic non-exclusive slip: branches that each *name a different way to cut* the level (e.g. **"By region" / "By segment" / "By quarter"** as siblings) instead of one axis's categories. It's deliberately conservative — it fires only when branches literally state differing axes up front, and never masks a concrete word-overlap.
- **Screen readers hear the tree.** The canvas now exposes real **ARIA tree semantics** — `role="tree"` on the board, `role="treeitem"` with **aria-level** (depth), **aria-expanded** (on branches with children), **aria-selected**, and an accessible name on every node — so assistive tech announces the structure that keyboard navigation already walks.
- **Multi-select & bulk actions.** Hold **⌘ / Ctrl / Shift and click** to select several nodes at once — each shows the selection ring — and a floating **action bar** sets their **status** or **priority**, or **deletes** them, all in one undoable step. **Delete** / **Backspace** now removes the whole selection. (The inspector still edits the primary node; box / rubber-band select is a follow-up.)
- **Works on a phone.** The editing workspace now has a real small-screen layout. Below ~640px the header collapses to **brand · MECE health · Export ▾ · ⋯** — Undo / Redo / Synthesis / Settings / Keyboard-shortcuts fold into the ⋯ menu — and the **inspector and review dock become a bottom sheet** that rises when you select a node (or open the review) and drops away when you deselect, so the canvas keeps the full width. The desktop layout is unchanged.
- **Save any tree as a reusable template.** **⋯ → Save as template…** banks the current tree's *structure* — labels, splits, dimensions — with values / evidence / status stripped, as a **custom template**. It appears under **Your templates** on the Templates page, ready to open as a clean starting point for the next engagement (how consultants actually reuse a diligence checklist or a standard tree).
- **Search across every tree.** The library search now matches **node labels and notes across all your trees**, not just their titles — so "which tree has the pricing-floor logic?" finds it, with a "matches …" hint on the card.
- **Navigate & review big trees.** The canvas gains a **minimap** (dots coloured by MECE health — amber = flagged, blue = high-priority) for wayfinding, and edges out of a flagged split now carry a **subtle always-on tint** so problem decompositions show without opening the dock. The **review dock** is now a triage tool: flags are **grouped by axis** (Overlaps / Gaps) and **ranked by branch priority** (80/20 first), each row shows its priority band, and every row has a **Review logic →** action that jumps to the inspector (not just CE gaps). And you can build & walk a tree from the **keyboard**: **Shift+Enter** adds a sibling, **↑/↓** move between siblings, **←** to the parent, **→** to the first child.
- **Per-tab MECE health.** With several trees open, each **tab** shows a health dot (green clean / amber to-review / grey not-yet-decomposed) so you can see at a glance which open trees still need work.
- **More ways to hand a tree off.** **Export ▾** gains **CSV (value model)** — one row per node (path, label, decomposition, ME/CE, priority, status, amount, unit, operator) so a value-driver tree opens straight in Excel — plus **Copy image** (the rendered tree onto your clipboard, for Slack or a slide) and a backend-free **Copy share link** (the tree packed into a `#doc=` URL that opens as a new tree on the other end). **PDF** and **PowerPoint** exports now carry a **title + date header**, so they read as a deliverable rather than a screenshot.
- **Import from outliners & mind-mappers.** **⋯ → Import outline…** now also accepts **OPML** (MindManager, OmniOutliner, Workflowy, Dynalist, …) alongside Markdown and a tree's JSON — nested `<outline>`s become the tree.
- **Close the keyless-AI loop.** The "suggest a split" prompt now asks the LLM for a **paste-ready Markdown outline**, and the inspector gains a **"paste the AI's split back"** box that grafts those sub-issues straight under the node — still no keys, no backend.
- **A governing answer, a verdict, and a one-page deliverable.** Answer-first problem solving starts with a **day-1 answer**: a slim **Answer** banner above the canvas captures the hypothesis the whole tree argues for. The **Answer-first synthesis** now opens with that answer and a rolled-up **verdict** from the top branches' status (e.g. _"3 of 5 top branches supported, 1 refuted — the answer partially holds"_), and the panel is **formatted** now, not a raw monospace block. A new **Export ▾ → Answer (1-page)** writes a clean, self-contained **HTML memo** — thesis on top, verdict, then the branches in priority order — the actual handoff, not a canvas screenshot.
- **Numbers in the synthesis.** The synthesis now carries each node's **value and unit**, the **rolled-up total** on a formula parent, and the **most-sensitive driver** — so a value-driver tree's answer includes its maths, and it all round-trips into Copy / Markdown.
- **Pick your first split when you build.** Building a tree from the hero question (or a **Try** chip) now opens a **“how do you want to split it?”** chooser — pick segment / binary / formula / … and you land on a scaffolded, already-checkable first split instead of a lone root box. **Start blank** keeps the old behaviour. (The hero always promised "scaffolds the first split"; now it does.)
- **A first-run canvas coach.** While a tree is still just its root, a dismissible tip on the canvas points at the two moves that start a MECE tree — press **Tab** to add a branch, then choose how it splits in the **Logic** panel to turn on the checks. Remembered per device once dismissed.
- **Quick add nests.** The **Quick add issues** box now builds levels: indent a line (Tab / spaces, or a bullet) and it becomes a sub-issue, so a whole multi-level decomposition drops in as one undoable step. A flat list still adds flat children.
- **Impact × ease priority matrix.** Set a branch's priority in one click on a 3×3 impact-by-ease grid — with its High / Medium / Low band shown live — instead of two separate rows. Press **P** on a selected node to bump its priority (none → low → medium → high).
- **Faster evidence entry.** Choose an evidence item's **strength** (anecdote / indicative / strong) *before* you add it, switch an item's strength with a direct picker (no more blind cycling), and **flip an item between supporting and contradicting** in place.
- **Richer print / PDF outline.** The print view now carries each node's **status, priority band, and evidence** — matching the Markdown export — so a printed or PDF'd tree no longer silently drops the analysis.
- **Native-dialog-free, safer delete.** Deleting a tree now asks in the app's own **confirm dialog** (styled as destructive, and separated from *New tree* by a divider in the ⋯ menu); renaming a tree uses an in-app prompt too — no more jarring browser `confirm` / `prompt` boxes.
- **Deeper MECE checks.** The engine now coaches the split types it can't *prove* exhaustive instead of showing a silent grey dot: a **process** split asks whether the stages run end to end (nothing before the first or after the last), and a **framework** split reminds you it organises thinking but isn't a provable partition — confirm nothing important sits outside. The **sibling-overlap** heuristic is sharper too: a word *every* sibling shares (with 3+ branches) is treated as the split's dimension, not an overlap, and a real overlap now **names the colliding pair** (and counts any others). And **formula** splits are checked for **double-counting** — a summed term named like a running total, or two terms with the same label, is flagged (product / difference terms stay clean by construction).
- **Name the dimension a split is cut on.** Each decomposition can carry the single **axis** it splits by (e.g. _by geography_) — set it in the inspector's **Logic** tab, with one-click common axes (customer / geography / product / time / stage). The named dimension shows on the node, in the answer-first synthesis, and in the AI-critique prompt. Naming it is how you keep a level MECE: one dimension per split.
- **MECE state is legible without colour.** The ME / CE indicators and the hypothesis-status border now carry a **glyph** (✓ / ! / – for pass / warn / unknown; ✓ / ✗ / ⊘ for supported / refuted / parked) plus a `title` and screen-reader label — so the product's core signal survives greyscale, colour-blindness, and assistive tech, not colour alone.

### Added
- **SVG export.** The **Export ▾** menu gains **SVG** — a true vector export of the canvas, alongside PNG / PDF / PPTX / JSON. Because an `.svg` is a document a browser executes when opened, the export is **sanitised at the sink**: the serialised markup is run through a new `sanitizeSvg` that strips any `<script>`, inline event handler (`on*`), and script-bearing URL (`javascript:` / `data:text/html`) before the file is written — and fails closed on anything unparseable. The export-safety guard was updated deliberately from “no SVG path” to “any SVG export must go through the sanitiser”, backed by unit tests proving scripts/handlers/URLs are removed.

### Fixed
- **The book links now open the book, not the app.** Opening *Issue Trees with MECE Studio* as PDF or EPUB — from **Learn MECE** or the **About** dialog — is a navigation request, and the service worker's `navigateFallbackDenylist` only excluded `.html`, so it rewrote the `.pdf`/`.epub` to `/index.html` and the app loaded instead of the file. (Standalone `.html` pages dodged it; the book links didn't.) `.pdf`/`.epub` are now on the denylist, so those navigations fetch the real file.
- **Deleting your last tree now clears it instead of silently re-creating it.** Removing the only tree — from a Start card or **⋯ → Delete tree** — used to immediately reseed a fresh `Why is this happening?` tree with the same default title, so it looked like the delete did nothing. Deleting the last tree now empties the library and returns you to the Start page's empty gallery, and an emptied library stays empty across reloads (only a genuine first run still seeds a starter tree).

### Added
- **Multi-document tabs.** Keep several trees open at once: opening or creating a tree adds a tab to a strip above the canvas (shown only when more than one is open). Click a tab to switch, **×** to close it (the tree stays in your library — closing the last tab returns to Start), and **+** to start a new one. The open set is remembered across reloads.
- **Quick add issues.** A new **⋯ → Quick add issues…** dialog: type one issue per line and add them all as children of the selected node (or the root) in a single undoable step — a fast way to dump a decomposition without clicking node by node. **⌘/Ctrl + Enter** adds.
- **Present & Print.** Two new ways to share a tree from the **⋯** menu. **Present** opens a full-screen, step-through presentation that walks the tree one decomposition at a time (depth-first) — each slide shows a question, its branches, and that split's MECE status; **← / →** (or Space) navigate and **Esc** exits. **Print…** opens a clean print preview that renders the whole tree as a nested outline (root question, kind + MECE summary, indented branches) with a **Print** button; a print stylesheet hides the app chrome so only the tree prints.
- **Import a tree — Markdown outline or JSON.** A new **⋯ → Import outline…** dialog takes a pasted **Markdown outline** (ATX headings and/or `-`/`*`/`+`/numbered bullets, nested by indentation) and builds a fresh tree — the first heading or line becomes the root question, everything else nests beneath it. It also accepts a tree's **JSON** (auto-detected), which runs through the schema migration runner and round-trips with the JSON export. Imports always open as a **new** library entry, so the current tree is never touched. (`domain/markdownImport.ts` + `services/import.ts`.)
- **JSON export.** The **Export ▾** menu gains a **JSON** option that downloads the raw document, round-tripping with **Open file…** — alongside the existing PNG / PDF / PPTX. Under the hood, the image/PDF/PPTX render logic moved out of the canvas into a dedicated `services/exporters/` folder (one lazy-loaded function per format), with JSON serialization now a single shared source of truth used by both export and file-save. (No SVG export: MECE deliberately rasterises every diagram export — see `export-safety.test.ts` — so a live-SVG path stays out by design.)
- **Open & save real `.json` files.** The overflow menu now has **Open file…**, **Save**, and **Save As…** backed by the File System Access API: opening a tree gives a real file handle, so a later **Save** writes straight back to the same file on disk — no re-download, no "downloads" clutter. The handle is remembered per tree in IndexedDB, so Save keeps working after a reload (re-prompting for permission when the browser requires it). Browsers without File System Access (Firefox, Safari) fall back automatically to a file-picker for open and a download for save. This sits **on top of** the localStorage autosave — your trees still persist locally as before; this just adds explicit file open/save. (`services/fileSystemAccess.ts` + `services/fileHandles.ts`.)
- **Schema migration framework — saved trees keep opening across versions.** A forward-only migration runner (`src/domain/migrations/`) now brings every persisted document up to the current schema before its shape is trusted, on each load and on file import. Migrations are pure, ordered, and registered one-version-at-a-time (`MIGRATIONS` → `migrateToCurrent`), with `CURRENT_SCHEMA_VERSION` tracking `SCHEMA_VERSION` so the two can't drift. The registry is empty today (MECE has had one schema), so legacy/unversioned saves are simply stamped to v1; a doc from an unknown *future* version is passed through untouched rather than crashing. Adding a real migration is now a single `vNToVN1.ts` file plus a registry row.
- **Faster first load — the editor loads on demand.** The Start landing no longer ships the editor: the whole workspace (the React Flow + dagre canvas, inspector, and clustered header) is code-split into its own chunk via `React.lazy`, dropping the eager first-load bundle from ~167 KB to ~100 KB gzipped. Start prefetches that chunk in the background once it has painted, so opening a tree stays instant. The size budget was tightened (168 → 115 KB) to lock the win in — a regression that re-bundles the canvas eagerly now fails the gate.
- **Bigger template library — named frameworks + more worked examples.** The **Templates** page gains a **Named frameworks** section: eleven classic strategy, marketing, and diagnosis frameworks as one-click starter trees — the **4Ps** and **Lauterborn 4Cs** marketing mixes, Ohmae's **3Cs**, **Porter's Five Forces**, **PESTEL**, **SWOT**, the **BCG** and **Ansoff** matrices, **McKinsey 7S**, the **AARRR** funnel, and a **fishbone (Ishikawa) 6 Ms** cause checklist. Each opens with its canonical branches filled in, and they're honestly typed as `framework` / `process` (never *provably MECE*) — the section says so, and the engine leaves exclusivity for you to confirm as you rename branches to your situation. The **Example trees** gallery gains six worked, multi-level trees: **market entry** (four logical gates), a fully-worked **M&A / "Chicken Express"** acquisition (most branches qualitative, but a provable synergy formula that must clear a $200M goal), **pricing** (cost floor / value ceiling / competitive anchor), top-down **market sizing** (formula splits that reconcile), **build vs buy vs partner**, and a **revenue value-driver** tree (revenue = customers × spend, every split reconciling). Both libraries stay registry-driven, so each new entry is a single catalogue row.
- **Editor redesign — the MECE differentiator, surfaced.** The flagged-split count is now a tree-level concern, not something you hunt for node by node. A **MECE health** chip in the header reads *✓ MECE clean* or *⚠ N to review* and opens a **review dock** listing every flagged split with its plain-language ME / CE reason, a one-click **Locate** (centres the node on the canvas — which dims the clean splits and amber-dashes the flagged edges while the dock is open), and a real **remedy** (add an "Other" bucket for segments, else a sub-issue — no fake "resolve", since warnings are computed). The **header** is reclustered into labelled groups — brand · the tree's title + a type badge (*Value-driver / Segmentation / … tree*) · health · undo/redo · Synthesis · a filled **Export ▾** menu · Settings · shortcuts · an **⋯ overflow** menu — and export moves off the canvas into that menu. The **inspector** is now **tabbed** (*Issue · Logic · Evidence · Value*), showing one facet at a time, with **Logic** opening automatically when the selected node's split needs review.
- **Installable app icons.** A proper maskable PWA icon (192 & 512, the brand issue-tree mark) so the install prompt and the home-screen / app-window show the MECE Studio logo instead of a generic glyph. Generated reproducibly from the favicon via `pnpm icons`.
- **Self-updating PWA.** When a new version is deployed, the running tab shows a non-intrusive **"A new version is available — Refresh now"** toast; clicking it swaps to the new build and reloads — never a silent reload, since you may have unsaved trees. The first visit shows a **"Ready to use offline"** toast, and **About → Check for updates** forces a check on demand. Built on `vite-plugin-pwa`'s `registerType: 'prompt'` flow with a small toast surface.
- **Start page.** A new landing surface — a workspace shell with a left sidebar (**Start, All trees, Recent, Templates, Needs review, Learn MECE**) and store-driven badges. A key-question hero builds a tree (or seeds one from an example-question chip); a **Templates** page surfaces every decomposition framework and example tree as a card — driven by the source registries, so a new framework or example appears with no edits; your saved trees show as cards with a **live MECE pill** that reads the same status the canvas does (*MECE clean* / *n* to check) over a mini tree-preview — and you can **rename, duplicate, or delete** a tree from its card. **Needs review** triages the trees with a flagged split. Press ⌘K to search trees; the workspace wordmark returns here from any tree.
- **Settings.** A new **⚙ Settings** panel (header), saved on your device, with three preferences — all defaulting to today's behaviour: **sort siblings by priority** (lay branches out highest-impact first instead of creation order, opt-in), **stricter overlap detection** (also flag shorter shared words between siblings, opt-in), and a **configurable formula tolerance** (how closely a value-driver split must reconcile to read as MECE — default 0.5%).
- **Keyboard shortcuts overlay.** Press `?` (or the new **?** button in the header) to open a cheat-sheet of every canvas shortcut; `Esc` or a click outside closes it.
- **Example trees.** Ready-made trees to open and learn from instead of starting blank — from the Start and Templates pages: a value-driver *operating profit* tree (formula branches that provably reconcile), a *customer churn* issue tree (segmented with an "Other" bucket, carrying hypotheses and evidence), and a *subscription launch* decision (built on a provably-MECE binary split). Each opens as a fresh tree in your library, so your own work is untouched.
- **The book.** A ~23,000-word practitioner’s book, *Issue Trees with MECE Studio* (`docs/guide/`, 11 chapters + 3 appendices), built to **PDF and EPUB** from one source by `pnpm book` — cover, clickable TOC, full document metadata, a pinned book id (so Kindle re-sends don’t duplicate), and an in-book issue-tree diagram generated from source constants so it can’t drift. A **Rebuild book** workflow regenerates + commits the artifacts when book inputs change (bot loop-guard) with an opt-in send-to-Kindle; both files ship to the site and are linked from the About dialog.
- **User manual.** A comprehensive, task-organized [`USER_GUIDE.md`](USER_GUIDE.md) covering every feature plus a full keyboard reference, rendered to a styled `/user-guide.html` and linked from the About dialog. Every catalogued feature is now marked documented, so the dashboard's manual-coverage reads 100%.
- **Dual licensing + About dialog.** Code ships under **Apache-2.0** (`LICENSE`); the book under **CC BY-NC 4.0** (`LICENSE-BOOK`). Third-party attribution + trademark notices live in `NOTICE.md`, rendered to a styled `/notices.html`. An in-app **About** dialog (header) carries the tagline, links to the user guide / notices / source, and the copyright + license summary.
- **Project dashboard + metrics.** A standalone dashboard (`/dashboard.html`) pairs a live GitHub repo pulse (commit frequency, types, authors, day/hour activity, recent commits & PRs) with CI-published project metrics — tests, coverage, bundle vs budget, code hygiene, churn/risk, and documentation coverage against a curated `docs/features.json` catalogue. `build-stats.mjs` + a **Stats** workflow refresh `stats.json` on every push; a `check-feature-coverage` gate validates the catalogue and flags CHANGELOG drift.
- **Multiple trees.** Keep a library of issue trees and switch between them from the **Start page**, where every tree is listed (named by its root question) as a card; **+ New** starts another and **Delete** removes the current one. Each tree persists under its own key; a previously single saved tree migrates into the library automatically, so nothing is lost.
- **AI assist (keyless).** Copy a ready-made prompt — _critique this tree's MECE_ (from the Synthesis panel) or _suggest a MECE split for this node_ (from the inspector) — with your tree embedded, to paste into Claude or ChatGPT. No API key, no backend.
- **PDF + PowerPoint export.** Export the canvas as a **PDF** or a **PPTX** slide from the header, alongside PNG. `jspdf` and `pptxgenjs` are loaded on demand (and reuse the PNG render), so they stay off the first-load bundle.
- **Reorder siblings.** Move a node up or down among its siblings from the inspector — arrange branches in the order that reads best.
- **Duplicate a subtree.** Copy a node and its whole subtree (with fresh ids) as a sibling from the inspector — handy for parallel branches that share a structure.
- **Find on the canvas.** A search box (top-left) rings every node whose label matches as you type and shows a match count; press Enter to zoom to the matches — handy in a big tree.
- **Collapse subtrees.** Any node with children gets a toggle — collapse to hide its subtree (the node shows a ▶ N badge with the hidden count) so you can focus on one branch of a big tree; the layout re-tidies around what's left visible. **Collapse all / Expand all** buttons (top-left) fold or unfold the whole tree at once.
- **Formula operators.** Choose how a formula split combines its children — **sum**, **product** (Price × Volume), or **difference** (Revenue − Cost) — so value-driver trees aren't limited to addition.
- **Sensitivity analysis.** On a formula split, a ranked readout shows how much the value swings when each leaf driver moves ±10%, one at a time — so the driver that matters most is obvious.
- **Inline editing + keyboard tree-building.** Double-click any node to rename it on the canvas (Enter / click-away commits, Escape cancels); with a node selected, **Enter / F2** edits it and **Tab** adds a child and edits it straight away — build a tree without leaving the keyboard.
- **Drag to re-parent.** Drag any node onto another to move it (and its whole subtree) under that node; the valid drop target is **highlighted** mid-drag so the result is predictable, then dagre re-tidies the layout. Auto-layout stays in charge — an invalid drop (onto the node's own subtree, the root, or empty space) isn't highlighted and snaps back.
- **Playwright e2e.** A real-browser test drives an actual drag to verify re-parenting + the drop-target highlight (`pnpm e2e`), and runs as a CI job. Browsers install project-local (`PLAYWRIGHT_BROWSERS_PATH=0`) so they sit under the AppLocker-allowed path.
- **Per-node notes.** Capture each issue's rationale, assumptions, or data sources in the inspector; a node carrying notes shows a small marker on the canvas, and notes flow into the Markdown export as an italic line under the bullet.
- **Open JSON.** Load a previously saved tree back in from the header (validated; the current tree stays in undo) — completing the save/open round-trip.
- **Units on values.** Set a unit (DKK, %, hrs, FTE…) beside a node's number — it shows on the node and in the Markdown export, and survives a value roll-up.
- **Keyboard shortcuts.** Undo / redo with Ctrl/⌘+Z and Ctrl/⌘+Y (or Shift+Z), and Delete / Backspace to remove the selected node — all ignored while typing in a field, so they never hijack the inspector inputs.
- **New tree.** Start over from a fresh question via the header — the previous tree is kept in undo, so nothing is lost.
- **M5 — PNG export.** Export the whole tree as a PNG image from the canvas; `html-to-image` is loaded on demand so it stays a separate lazy chunk off the first-load bundle. The size budget now measures only the eager entry chunk(s).
- **M4 — value-driver roll-up.** On a formula split, one click rolls the children's numbers up into the parent's value (sum / product / difference); the reconciliation check then confirms the totals add up.
- **M4 — hypothesis status.** Mark any node open / supported / refuted / parked; the node gets a colour-coded left edge and the synthesis flags ✓ supported, ✗ refuted, ⊘ parked.
- **Export.** Copy the tree as a Markdown outline (each split's MECE state annotated inline) or save the full document as JSON, from the header.
- **M3 — evidence + answer-first synthesis.** Attach supporting/contradicting **evidence** (with a cyclable strength) to any node — nodes show ✓/✗ counts. A new **Synthesis** panel reads the tree answer-first: it leads with the highest-priority branch, lists branches in priority order, surfaces each node's evidence, and flags MECE gaps/overlaps — copyable as Markdown.
- **M3 — branch prioritisation.** Mark each issue's **impact × ease** in the inspector; the node shows a priority-band chip (High / Medium / Low, from the 1–9 score) so the branches worth chasing first stand out. (Leaf evidence and answer-first roll-up still to come.)
- **M2 — deeper MECE rules + decomposition scaffolds.** The engine now evaluates each axis separately: binary (A / not-A) and formula stay provably MECE; **segments** are collectively exhaustive only with an explicit "Other" bucket; other split types get a **sibling-overlap heuristic** that flags siblings sharing a content word (with generic/placeholder nouns ignored, so fresh splits don't flag themselves). `decompose()` **scaffolds** type-appropriate starter children (binary → A / not-A, segments → two + Other, process → stages, formula → terms), surfaced as "Decompose by …" buttons on leaf nodes.
- **M1 — interactive issue-tree canvas.** A React Flow + dagre (left-to-right) canvas that renders the tree from the store: select a node, add sub-issues, choose how it decomposes (freeform / segments / process / binary / formula / framework), edit labels and numeric values, and delete subtrees. The MECE engine runs live on every change — each decomposition shows mutually-exclusive / collectively-exhaustive status as coloured dots on the node and as explained warnings in the inspector. Undo/redo, localStorage persistence, and auto-fit on tree changes. 23 unit tests across the domain core + projection.
- **Project scaffold (M0).** React 19 / Vite 8 / TS 6 / Tailwind 4 / Zustand 5 / vite-plugin-pwa. Domain model (`IssueNode`, `Split` as the MECE unit, `IssueTreeDoc`) + `factory`, a minimal app shell + Zustand store seeded with a root question, and the GitHub Pages deploy pipeline. Live at <https://mece-studio.struktureretsundfornuft.dk>.
- **Local gate + CI + working agreement.** `pnpm verify` runs the full fail-fast gate — typecheck → lint/format (Biome) → dead-code (knip) → tests (Vitest) → build → bundle-size budget — via shell-free node runners (`scripts/verify.mjs`, `scripts/build.mjs`) that work under the local AppLocker policy and on CI. CI runs the same gate. `CLAUDE.md` captures the working agreement; `bundle-budget.json` is the single source of truth for the size budget.

### Changed
- **Richer Markdown export.** Copying a tree as Markdown now includes each node's hypothesis status, priority, and evidence — alongside the value, MECE state, and notes it already carried — so a pasted outline holds the whole analysis. The keyless AI-assist prompts (which embed the same Markdown) get the extra context too.
- **Calmer auto-fit.** The canvas now re-centres only when a new or changed node would land outside the current view — so editing a large tree no longer yanks the whole thing back to a full fit on every change, while a node added off-screen still pulls into view. A freshly-opened tree still fits to the window.

### Fixed
- **Cleaner undo history.** Editing a node's label, value, or notes and leaving it unchanged (e.g. clicking into a field and back out) no longer adds a redundant step to undo/redo.
- **Accessible dialogs.** The About and keyboard-shortcuts dialogs now move keyboard focus into the dialog when it opens, keep `Tab` cycling inside it, and restore focus to the button that opened it on close — so keyboard and screen-reader users aren't dropped behind the modal.
- **Reproducible book builds.** `pnpm book` now produces a byte-identical PDF and EPUB for unchanged sources, so a rebuild — or a send-to-Kindle dispatch — no longer commits churn. Pinned the PDF's trailer `/ID` (Chromium randomises it per run), derived every timestamp from the last book-content commit, and stamped jszip's auto-created folder entries (which had defaulted to the build clock).
- **Node styling warning.** The issue node mixed the `border` shorthand with a `borderLeft` status override, so React warned on every rerender. It now uses longhand border props — identical look, clean console.
